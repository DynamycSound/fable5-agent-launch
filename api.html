● API QUICKSTART

Put Fable in your product.

The Fable API lets you create an agent workflow, check its state, and receive progress events. Base URL: https://api.fable5.dev/v1

Authenticate

Pass your workspace API key as a bearer token. Keep it on your server; never expose it in client-side code.

bash
export FABLE_API_KEY="fbl_live_..."
curl https://api.fable5.dev/v1/me -H "Authorization: Bearer $FABLE_API_KEY"

Create a workflow

Send an objective and repository reference. The response includes a workflow ID you can poll for status or subscribe to events.

bash
POST /v1/workflows
{ "objective": "Add CSV export", "repository": "acme/web" }

Workflow events

Configure a webhook for workflow.started, workflow.blocked, and workflow.completed. Verify each delivery against the Fable-Signature header.

Production note

These endpoints are reference documentation for the Fable 5 launch site. Use test keys while prototyping and rotate production credentials through your secret manager.
